# City Sentinel — NexaCity OS v3.2

> Умная система мониторинга городской дорожной инфраструктуры Казахстана.
> Отслеживает трафик, инциденты и состояние дорог **20 городов** в реальном времени.

---

## Нетехническое описание

### Что это?

**City Sentinel** — это веб-платформа для мониторинга и управления дорожной обстановкой в городах Казахстана. Система показывает в реальном времени:

- **Трафик** — баллы пробок от 0 до 10, средняя скорость потока, тренд загруженности за 12 часов.
- **Инциденты** — ДТП, дорожные работы, перекрытия с привязкой к реальным улицам на карте.
- **Состояние дорог** — процент износа основных магистралей каждого города.
- **ИИ-аналитику** — автоматические рекомендации по управлению трафиком: перенаправление потоков, корректировка светофоров, планирование ремонта.
- **Live-события** — инциденты, появляющиеся и исчезающие в реальном времени через WebSocket-соединение.

### Какие города поддерживаются?

Алматы, Астана, Шымкент, Актобе, Караганда, Павлодар, Атырау, Костанай, Тараз, Орал, Семей, Усть-Каменогорск, Петропавловск, Кокшетау, Талдыкорган, Туркестан, Кызылорда, Актау, Жезказган, Экибастуз.

### Для кого это?

- **Городские администрации** — оперативный контроль дорожной обстановки, принятие решений на основе данных.
- **Диспетчерские центры** — мониторинг инцидентов, рассылка оповещений, координация экстренных служб.
- **Аналитики и урбанисты** — изучение паттернов загруженности, планирование инфраструктуры.
- **Граждане** — информирование о пробках, авариях и перекрытиях.

### Как пользоваться?

1. Откройте лендинг `/` — ознакомьтесь с возможностями системы.
2. Перейдите на страницу авторизации `/auth` (есть демо-аккаунт — один клик).
3. Попадёте на **Главный дашборд** `/dashboard` — выберите город в боковом меню.
4. Смотрите **KPI-карточки**, **графики**, **карту с полилиниями трафика** и **маркерами инцидентов**.
5. Нажмите на маркер инцидента — откроется **ИИ-анализ** с рекомендацией.
6. Перейдите на **Карту городов** `/dashboard/map` — интерактивная карта всего Казахстана с выбором города.
7. В **Настройках** `/dashboard/settings` — профиль, уведомления, тема, язык, интервал обновления.

## Техническое описание

### Архитектура

Проект состоит из двух независимых частей:

```
city-sentinel/
├── frontend (React SPA)         — Vite + React 18 + TypeScript
├── backend  (Python API)        — FastAPI + Uvicorn
└── WebSocket канал              — real-time push events
```

**Фронтенд** собирается Vite и деплоится на Vercel (SPA-режим с fallback на `index.html`). **Бэкенд** запускается отдельным сервером и предоставляет REST API + WebSocket. Связь между ними — через `VITE_API_BASE_URL` (по умолчанию `http://127.0.0.1:8080`).

#### Realtime (WebSocket)

Хук `useRealtimeEvents` (`src/hooks/useRealtimeEvents.ts`) подключается к `ws://.../ws/events/{city_id}`:
- При подключении получает **snapshot** всех активных событий.
- Каждые ~3 секунды получает **update** с новыми и истёкшими событиями.
- Новые события подсвечиваются "LIVE"-бейджем на 5 секунд.
- Автоматический реконнект через 3 секунды при обрыве.

#### Данные фронтенда

- `src/lib/traffic.ts` — типы, маппинги городов (названия, координаты), API-функции `fetchTraffic()` и `fetchMapData()`.
- `src/data/kazakhstan-border.ts` — GeoJSON-полигон границы Казахстана для отображения на карте.

#### REST API эндпоинты

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/traffic/{city_id}` | Полные данные трафика: score, инциденты, дороги, тренд, ИИ-рекомендация |
| `GET` | `/api/map-data/{city_id}` | GIS-данные: полилинии с congestion level, маркеры инцидентов с таймстампами и ИИ-рекомендациями |
| `GET` | `/api/cities` | Список всех 20 городов с метаданными |
| `GET` | `/health` | Статус системы, версия, кол-во городов |

#### WebSocket эндпоинты

| Путь | Описание |
|---|---|
| `/ws/events/{city_id}` | Стрим событий одного города (snapshot → periodic updates) |
| `/ws/events` | Стрим событий ВСЕХ городов (для обзорных дашбордов) |

**Протокол:**
- `snapshot` — при подключении, все активные события.
- `update` — каждые ~3 сек: `new` (новые события), `expired` (удалённые ID).

#### Движок симуляции

Система генерирует **реалистичные данные** без подключения к внешним API (режим по умолчанию, `simulation_enabled=True`):

- **Детерминированность** — `_seeded_rng(city_id)` использует SHA-256 хеш от `city_id + time_bucket(5 мин)`, поэтому данные стабильны при повторных запросах в пределах 5-минутного окна.
- **Время суток** — загруженность следует кривой rush-hour (пики в 08:30 и 17:30, минимум в 03:00).
- **Масштаб города** — population_factor (0.3–1.0): крупные города = больше инцидентов, выше пробки.
- **6 типов событий** — ДТП, ремонт, перекрытие, затор, погода, полиция (с весами вероятности).
- **4 уровня severity** — low, medium, high, critical (с весами).
- **TTL** — каждое событие живёт от 3 мин (затор) до 4 часов (ремонт), затем автоматически удаляется.
- **Геометрия** — события привязаны к реальным координатам дорог из OSRM-кэша.

#### ИИ-советник с RAG + ChatGPT (`ai_advisor.py` + `openai_advisor.py` + `rag_context.py`)

Система использует **Retrieval-Augmented Generation (RAG)** с OpenAI ChatGPT для формирования аналитических выводов:

**Архитектура RAG-пайплайна:**
1. `rag_context.py` — **RAG Context Builder**: сериализует все реальные данные города (метрики, инциденты, состояние дорог, тренды, дорожную сеть) в структурированный текстовый документ (~2000 символов).
2. `openai_advisor.py` — **ChatGPT Client**: отправляет RAG-документ в OpenAI Chat Completions API с строгим system prompt и JSON-mode.
3. `ai_advisor.py` — **Orchestrator**: пытается ChatGPT → при ошибке падает на детерминированный rule-based engine.

**Меры против галлюцинаций (anti-hallucination):**
- LLM получает **ТОЛЬКО** верифицированные данные из пайплайна (никакого web search, никакой памяти).
- System prompt **явно запрещает** выдумывать улицы, числа или события, отсутствующие в отчёте.
- `temperature=0.3` минимизирует креативный дрейф.
- `response_format={"type": "json_object"}` — принудительный JSON-вывод.
- Ответ **валидируется** через Pydantic перед возвратом клиенту.
- При любой ошибке — автоматический **fallback** на rule-based engine.

**Выходные данные (`AIInsight`):**
- **Текстовое резюме** — ситуация в городе, основанная на реальных данных.
- **Приоритет** — HIGH / MEDIUM / LOW.
- **Список действий** — 3–5 конкретных рекомендаций со ссылками на реальные улицы.
- **Уверенность** — 0.0–1.0, самооценка модели на основе полноты данных.
- **model_version** — `chatgpt-gpt-4o-mini` или `rule-engine-v3.2` (для прозрачности).

**Также:** каждый инцидент на карте получает персональную AI-рекомендацию от ChatGPT (или fallback-шаблон).

#### Дорожная геометрия (`road_geometry.py` + `osrm_router.py`)

Для каждого из 20 городов вручную определены вейпоинты основных дорог (от 4 до 25 дорог на город). OSRM-роутер:
1. Берёт вейпоинты → отправляет в OSRM API → получает плотную полилинию, повторяющую реальную геометрию дороги.
2. Кэширует результат в `_geometry_cache/{city_id}_roads.json`.
3. При последующих запросах читает из кэша (warm cache для всех 20 городов предзаполнен).

#### Подготовка к реальным API

В `traffic.py` заложены stub-функции для:
- **Yandex Traffic API** — реальный балл пробок.
- **2GIS API** — дорожные сегменты, геокодинг.

Если в `.env` указаны `YANDEX_TRAFFIC_API_KEY` или `TWOGIS_API_KEY`, система попытается использовать реальные данные, с fallback на симуляцию.
